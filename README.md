## SAP Customer Experience (SAP CX)

Nuestra web presenta una adaptación sobre información permitida free sobre la plataforma SAP Customer Experience (SAP CX); una solución CRM diseñada para ayudar a las empresas a gestionar de forma integrada sus ventas, marketing, servicio al cliente y comercio digital. El sitio explica cómo SAP CX explica las características esenciales de SAP CRM Costumer Experience,muestra las principales funcionalidades, beneficios y módulos de SAP CX, los datos de contacto, tablas con respecto a otros CRM, así como información spermitida sin registro de los precios obtenidos.

   ## Guía de Uso de la Web sobre SAP Customer Experience (SAP CX)

Esta web ofrece una recopilación de información pública y permitida sobre **SAP Customer Experience (SAP CX)**. A continuación, se describe cómo navegar en cada sección:

### 1. Inicio (START)
Presenta de forma breve una visión general de SAP CX y explica el propósito de la web. Es el punto de partida para comprender la plataforma y su ecosistema.

### 2. Características y Funcionalidades (FEATURES)
Incluye una descripción sencilla de las funciones principales de SAP CX, como:
- Ventajas
- Personalización de la información
- Soluciones integradas
- Ayuda mediante análisis 

Esta sección permite entender rápidamente el alcance del CRM.

### Módulos de SAP CX 
Resumen de los módulos más importantes de SAP CX, explicando su papel dentro de la plataforma y cómo contribuyen a la experiencia del cliente.

### Comparativas con Otros CRM
Tablas que muestran las diferencias entre SAP CX y otros CRM del mercado. Útil para evaluaciones rápidas y decisiones informadas.

### 5. Información Pública de Precios (PRICING)
Incluye los precios disponibles sin registro, organizados de forma clara para facilitar su consulta.

### 6. Datos de Contacto (SUPPORT)
Proporciona información para comunicarse con el equipo responsable o solicitar más detalles relacionados con SAP CX.

## Maquetación (Requerimientos)

## 1. HTML Semántico

- Estructura con etiquetas HTML5 (header, nav, main, section, footer).
- Código limpio, ordenado y con jerarquía correcta de títulos (h1–h6).

## 2. Diseño Responsive

- Uso obligatorio de Flexbox para layouts.
- Media Queries para desktop, tablet y móvil.
- Imágenes, videos y tablas.
- Estilos y Tipografía
- Uso de unidades relativas (rem, em, %, vw, vh).
- Paleta de colores consistente y definida.

## 3. Componentes y Contenedores

-Cajas con separación interna (padding) y alineación center o space-between según el caso.
-Botones homogéneos (misma tipografía, colores y hover states).
T-ablas limpias y legibles, con bordes suaves o alternancia de fondo.
-Texto con contraste adecuado.
-Imágenes con alt.
-Elementos interactivos accesibles por teclado.

## 4. Rendimiento y Buenas Prácticas

- CSS modular (por componentes o secciones).
- Selectores simples y optimizados.
- Cargas multimedia optimizadas.
